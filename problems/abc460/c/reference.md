# abc460 c 解説の材料

## 公式解説
URL: https://atcoder.jp/contests/abc460/editorial/21026

### 問題概要
シャリが N 個(重さ A_i)、ネタが M 個(重さ B_j)ある。1 つのシャリと 1 つのネタを組み合わせて寿司を作るには
「ネタの重さ ≦ シャリの重さ × 2」を満たす必要がある。各シャリ・ネタは 1 回しか使えないとき、
作れる寿司の最大個数を求める。

### 想定解法(尺取り法)
1. A, B をそれぞれ昇順にソートする。
2. 軽いネタから順に処理していき、まだ使っていないシャリの中で条件 `2*A ≧ B` を満たす
   **最小の**シャリを割り当てる、という貪欲が最適(軽いネタに軽いシャリを使っておくほど、
   重いシャリを後の重いネタのために温存できるため損をしない)。
3. 実装は二本のポインタで O(N + M) で行える。
   - `idx`: シャリ側のポインタ(まだ使っていない先頭)
   - ネタを昇順に見ながら、`a[idx]*2 < b[i]` の間は `idx` を進めて条件を満たさない
     (小さすぎる)シャリを読み飛ばす。
   - 条件を満たすシャリが見つかれば 1 組成立、`idx` を進めて次のネタへ。
4. ソートが O((N+M) log(N+M))、尺取り部分は O(N+M) なので全体もそれで抑えられる。

## 詰まったポイント
提出コード(`main.cpp`)は次のようになっていた。

```cpp
int tmp = min(n, m);
int idx = 0;
for (int i = 0; i < tmp; i++)
{
    if (a[i] * 2 >= b[idx])
    {
        ans++;
        idx++;
    }
}
```

- ループの上限が `min(n, m)` になっており、**シャリ配列 `a` を `i`(=ループ変数そのもの)で
  逐次読んでいる**。つまり「条件を満たさなかった `a[i]` を読み飛ばして次の `a` を試す」
  つもりでも、`i` は毎回必ず 1 つずつしか進まず、`n > m` のとき配列の後半にある
  (十分重い)シャリまで `i` が到達できないことがある。
- 具体例: `N=3, M=2`, `A=[1,1,100]`, `B=[2,3]`。
  正しい答えは 2 組(A[0]=1↔B[0]=2, A[2]=100↔B[1]=3)だが、
  `tmp = min(3,2) = 2` なのでループが `i=0,1` しか回らず、`A[2]=100` が一切見られないまま
  終了し、出力が 1 になってしまう(尺取りの「使っていないシャリ全体」を
  ポインタで独立に管理できていなかったのが原因)。
- 教訓: 尺取り法で「片方の配列を読み飛ばしながら進める」ときは、
  読み飛ばし用のポインタと外側ループのポインタを**同じ変数にしない**こと。
  今回のように片方の配列の要素数が多い場合、外側ループの上限を
  `min(n, m)` にすると後半の要素に手が届かなくなる。

## 解き直し後のACコード

```cpp
#include <bits/stdc++.h>
using namespace std;

int main()
{
    cin.tie(nullptr);
    ios::sync_with_stdio(false);

    int n, m;
    cin >> n >> m;
    vector<long long> a(n), b(m);
    for (int i = 0; i < n; i++) cin >> a[i];
    for (int i = 0; i < m; i++) cin >> b[i];
    sort(a.begin(), a.end());
    sort(b.begin(), b.end());

    long long ans = 0;
    int idx = 0; // まだ使っていないシャリの先頭
    for (int i = 0; i < m; i++)
    {
        while (idx < n && a[idx] * 2 < b[i]) idx++; // 小さすぎるシャリを読み飛ばす
        if (idx < n)
        {
            ans++;
            idx++;
        }
    }

    cout << ans << endl;
    return 0;
}
```

サンプル1で動作確認済み(出力 `5` が `sample-1-expected.txt` と一致)。
また上記の反例(`3 2 / 1 1 100 / 2 3`)でも正しく `2` を出力することを確認済み。
